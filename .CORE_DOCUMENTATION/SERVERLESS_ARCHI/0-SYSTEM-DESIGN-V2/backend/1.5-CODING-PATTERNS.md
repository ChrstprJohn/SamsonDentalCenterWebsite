# 5️⃣ Golden Coding Patterns & DRY Blueprints

To enforce strict separation of concerns, improve serverless performance (cold starts), and maximize DRY (Don't Repeat Yourself), developers must follow these functional, Zod-first coding templates.

## A. Pure Zod Transformation Blueprint

Define `DatabaseSchema` (raw) and `ResponseSchema` (transformed) in one file.
**Benefit:** Zero manual mapping code. Zod handles the snake_case to camelCase conversion.

```ts
// src/modules/services/dtos/management/service-response.dto.ts
import { z } from "zod";

// 1. Raw Database Shape (snake_case)
const serviceDbSchema = z.object({
  id: z.string().uuid(),
  name: z.string(),
  duration_minutes: z.number().int().positive(),
  is_active: z.boolean().default(true),
});

// 2. Application Shape via Transform (camelCase)
export const serviceResponseSchema = serviceDbSchema.transform((data) => ({
  id: data.id,
  name: data.name,
  durationMinutes: data.duration_minutes,
  isActive: data.is_active,
}));

export type ServiceResponseDto = z.infer<typeof serviceResponseSchema>;
```

## B. Functional CQRS Repository Blueprint

Export functions like `export const createServiceCommand = (supabase: SupabaseClient) => async (data) => ...`.
**Benefit:** Better cold-start performance and easier testing than OOP classes. Parses DB results through the unified Zod pipeline.

```ts
// src/modules/services/repositories/management/service.commands.ts
import { SupabaseClient } from "@supabase/supabase-js";
import { CreateServiceDto } from "../../dtos/management/create-service.dto";
import { ServiceResponseDto, serviceResponseSchema } from "../../dtos/management/service-response.dto";

export const createServiceCommand = (supabase: SupabaseClient) => {
  return async (data: CreateServiceDto): Promise<ServiceResponseDto> => {
    const dbPayload = {
      name: data.name,
      duration_minutes: data.durationMinutes,
      is_active: data.isActive,
    };

    const { data: result, error } = await supabase
      .from("services")
      .insert([dbPayload])
      .select()
      .single();

    if (error) throw new Error(`Failed to create service: ${error.message}`);
    
    // Parse through Zod pipeline (handles snake_case to camelCase transformation)
    return serviceResponseSchema.parse(result);
  };
};
```

## C. Functional Use Case Blueprint

Pure business logic functions that receive dependencies (like `createService` command function) as arguments.
**Benefit:** No `this` context or constructor overhead. Functional Dependency Injection.

```ts
// src/modules/services/use-cases/management/create-service.use-case.ts
import { CreateServiceDto } from "../../dtos/management/create-service.dto";
import { ServiceResponseDto } from "../../dtos/management/service-response.dto";

export const createServiceUseCase = (
  createService: (data: CreateServiceDto) => Promise<ServiceResponseDto>
) => {
  return async (data: CreateServiceDto): Promise<ServiceResponseDto> => {
    // Pure business validation rules go here
    return await createService(data);
  };
};
```

## D. Streamlined Server Action Blueprint

A clean, 3-step execution: 1. Parse Input, 2. Get Client & Inject Dependencies, 3. Call Use Case.
**Benefit:** High readability and strictly declarative flow.

```ts
// src/modules/services/actions/management/create-service.action.ts
"use server";

import { CreateServiceDto, createServiceSchema } from "../../dtos/management/create-service.dto";
import { createServiceUseCase } from "../../use-cases/management/create-service.use-case";
import { createServiceCommand } from "../../repositories/management/service.commands";
import { createClient } from "@/shared/database/server";

export async function createServiceAction(data: CreateServiceDto) {
  try {
    // 1. Zod input validation
    const parsed = createServiceSchema.parse(data);

    // 2. DI Setup (Functional)
    const supabase = await createClient();
    const repoCommand = createServiceCommand(supabase);
    const useCase = createServiceUseCase(repoCommand);

    // 3. Execution
    const result = await useCase(parsed);
    return { data: result };
  } catch (error: any) {
    return { error: error.message || "Failed to create service" };
  }
}
```
